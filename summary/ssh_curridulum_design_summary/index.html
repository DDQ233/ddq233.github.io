一、课程设计要求

题目举例：

  • 华软新生注册系统的设计与实现（易）
  • 华软教材订购系统的设计与实现（易）
  • 华软教学管理系统的设计与实现
  • 华软二手交易平台的设计与实现
  • 华软图书管理系统的设计与实现
  • 华软宿舍管理系统的设计与实现
  • 华软课室申请系统的设计与实现
  • 华软校园快递系统的设计与实现
  • 华软学籍管理系统的设计与实现
  • 计算机系资料收集系统的设计与实现（学生奖惩信息管理，学生社会实践活动管理，教师科研信息管理，教师奖惩信息管理）

具体要求：

1. 独立完成

2. 技术要求：

2.1 运用所学的 Spring Boot 、 Spring Cloud 、 SSM 等相关技术实现，必须要用到微服务的技术。

2.2 页面运用相应的框架技术实现，如 X-Admin 、 LayUI 、 EasyUI 等。

3. 功能要求

  • 有增删改查功能，并能进行分页显示
  • 有文件上传功能，并对上传文件大小和类型进行限制
  • 数据库操作必须用到关联关系
  • 有拦截器、验证功能

二、题目与主题

题目： 物联网设备管理系统

主题和目标： 为了学习和巩固Spring、Spring MVC、MyBatis和SpringBoot框架，做了一个简单的物联网设备管理系统。此系统能够实现管理员与用户的登入与登出，管理员能够对设备进行增删改查，同时管理员也可以修改个人信息。用户可以对设备进行增删改查。

七、开发过程中遇到的难题与解决办法

1. Thymeleaf 技术不熟悉，标签使用不熟练

        因为没有前端 HTML/CSS/JavaScript 技术的基础，对于标签，元素等组件不熟悉，导致了学习和使用 Thymeleaf 技术时产生了困难，有些东西看不懂，不会用。

        解决办法：了解前端页面常用到的标签、元素、组件、表单等组件的使用，补充基础，对基础理解之后，再进行对 Thymeleaf 技术的学习，不断往复学习与实践。

2. MyBatis 与 Spring Boot 框架整合不熟练

        在做课程设计的过程中，参考了课程以往的实验，巩固 MyBatis 和 Spring Boot 框架整合的知识。

3. Dao 层屡屡报错

        因为课程设计中有一些增删改查的功能是相同或者相类似的，于是就先做好一份 Mapper 映射文件，以此为标准，再复制一份作为其他 Mapper 映射使用，为此想要提升开发效率，简化操作，这样的操作为开发过程中 Dao层 屡屡报错埋下伏笔。相类似功能的 Mapper映射文件基本都是以第一份为基础的 Mapper 映射文件，修改 Mapper 映射文件的过程中需要添加或删除功能。因此，“标准Mapper映射文件”某些地方的 SQL 语句在语法上出现了细节性的错误，导致后续开发中没有发现，直到项目编译运行时才发现问题，这时已经积累了大量的 Bug，并且在开发 Dao层 的过程中更换过项目包结构，导致问题进一步增加，Debug难度进一步加大，需要不断地修改调试。这个问题在开发的过程中占总的 Debug 时间的一半，需要引起重视。

解决办法的流程：

  1. 检查运行错误信息。
  2. 检查错误类型。
  3. 检查错误位置。
  4. 读懂错误信息的具体内容。
  5. 定位错误位置。
  6. 修改错误。

        如果下次还遇到这种情况，可以先直接检查 Mapper映射文件，应该可以一次性把问题都解决。

检查内容：

  1. 检查命名空间。
  2. 检查 SQL 语句的语法
  3. 检查 SQL 标签的返回类型与参数类型。
  4. 检查 SQL 标签ID。
  5. 检查 XML 标签。
  6. 检查 Mapper 映射文件是否被正常编译放到 target 目录下
