一、课程设计要求

总体设计参考：


网关部分要求：

  1. 使用 Linux 实现一个简易的物联网网关。（Linux 使用 WSL 或者 VMware虚拟机）
  2. 能够从串口接收数据，终端节点可选择实物硬件、Proteus仿真、PC机上的串口调试助手。
  3. 能够把串口接收的传感器数据保存到 MySql 数据库。
  4. 能够把串口接收的传感器数据通过网络（TCP、UDP、MQTT等协议）发送给客户端。
  5. 能够从网络（TCP、UDP、MQTT等协议）中接收数据指令并通过串口发送到终端节点驱动外设（LED灯、电机等）的工作。
  6. 能够运用多线程的知识，从终端节点上行的数据处理在主线程中运行，下行的数据处理在子线程中运行。

3. 工作流程

3.1 传感器数据传输

a. 底层硬件采集数据发送到网关：

DHT11 温湿度传感器周期性采集数据，通过单总线协议，给 Arduino 单片机传输数据，单片机接收到数据以后整合成 JSON 数据格式的数据，通过串口发送给网关。


b. 网关保存数据到云端数据库

网关接收从串口发送过来的 JSON 数据，把温度与湿度解析出来，保存到数据库中，然后通过 MQTT 协议在 /sensor/dht11 主题发布有关数据的消息。


c. JavaWeb应用读取最新数据并展示

JavaWeb 应用前端页面使用 JavaScript 和 Ajax 技术，设置定时器定时请求能获取最新数据的 API ，获取数据后进行解析并刷新图表展示的数据。


3.2 硬件设备控制

控制设备：

在 JavaWeb 应用中，在请求 API 的处理中添加 MQTT 消息发布功能，在 /ctrl 主题上发布消息，网关收到消息并处理，通过串口往单片机发送指令，单片机解析指令控制电机。


四、开发过程中遇到的难题与解决办法

1. 在网关中通过串口获取到的数据不完整

解决办法：单片机编代码把数据整合成 JSON 数据格式的数据，数据发送的延时适当延长。

2. 网关使用多线程、多 MQTT 实例，出现消息发送失败的问题

在每一个线程中都有一个 MQTT 连接实例，当从串口接收到数据之后就往主题推送消息，但偶尔有一段时间会推送失败，原因不明，执行发送函数返回的错误代码是 -3 ，源码解释为 MQTT Client 已断开连接。

解决办法：使用 while 循环，不断推送数据直到数据发送成功为止。
